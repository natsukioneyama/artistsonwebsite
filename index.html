About GitHub – How It Works & Who Owns the Site

GitHub is a Microsoft-owned platform and one of the largest code-hosting services in the world. It is where millions of developers and designers store, share, and publish their projects. In terms of reliability, speed, and stability, it is one of the strongest free hosting environments available today.

What is “hosting”?

Hosting simply means placing your site files (HTML / CSS / JavaScript, etc.) on an online server so anyone can access your website. Many people imagine paid servers or monthly contracts, but GitHub Pages provides hosting completely for free. Creating an account is also free.

Why many professional designers use GitHub today

In recent years, more and more professional web designers have started building client websites on GitHub Pages. The reason is simple — it meets all the essential conditions:

  • Fast
  • Stable
  • Secure
  • No running costs
  • Hard to break
  • No dependency (the owner is always you)

How this differs from design studios and standard freelancers

Even if a designer uses GitHub, most studios or freelancers will still charge “maintenance fees” or “management fees” every month or every year.

But in my system, the site is built directly inside your own GitHub account, and the code is 100% owned by you. As long as your GitHub account exists, the site remains your asset forever.

Why this method is ideal for artist portfolios

Portfolio sites do not need heavy features or large servers. In reality, the best portfolio is one that is:

  • Lightweight
  • Stable and hard to break
  • Fast
  • Easy to maintain
  • Independent of any designer or company

Using this structure allows me to create a long-lasting, agency-level portfolio site with minimal cost and maximum control for the artist.

À propos de GitHub – Fonctionnement et propriété du site

GitHub est une plateforme appartenant à Microsoft et l’un des plus grands services d’hébergement de code au monde. Qu’est-ce que l’« hébergement » ?

Héberger un site signifie simplement placer les fichiers (HTML, CSS, JavaScript, etc.) sur un serveur en ligne pour que n’importe qui puisse y accéder. Beaucoup de personnes imaginent qu’il faut payer un serveur ou un abonnement, mais GitHub Pages propose un hébergement entièrement gratuit. La création de compte est également gratuite.

Pourquoi de nombreux designers professionnels utilisent GitHub aujourd’hui ?

Depuis quelques années, de plus en plus de web designers construisent les sites de leurs client·e·s sur GitHub Pages. La raison est simple : GitHub remplit toutes les conditions essentielles.

  • Rapide
  • Stable
  • Sécurisé
  • Aucun coût d’hébergement
  • Difficile à « casser »
  • Pas de dépendance (le/la propriétaire, c’est toujours vous)

Ce qui distingue cette méthode des studios ou freelances classiques

Même lorsqu’un designer utilise GitHub, la plupart des studios ou freelances facturent tout de même des « frais de maintenance » ou de « gestion » chaque mois ou chaque année.

Dans mon système, le site est construit directement dans votre propre compte GitHub, et le code vous appartient à 100 %. Tant que votre compte GitHub existe, votre site reste votre propriété.

Les portfolios n’ont pas besoin de fonctionnalités lourdes ou de grands serveurs. En réalité, un bon portfolio doit être :

  • Léger

FAQ

Est-ce un site entièrement sur mesure ?

La base est un site semi-personnalisé, mais le résultat final est proche de la qualité d’un site entièrement sur mesure.

Quelle est la différence entre “full custom” et “semi custom” ?

Le semi-custom classique utilise des modèles existants ou un CMS (Wix, WordPress, etc.) en ajustant seulement les couleurs ou quelques éléments de mise en page. Mon approche n’utilise aucun template. La mise en page, la structure et la présentation visuelle sont créées entièrement à partir de zéro — une approche très proche du “full custom”.

Pourquoi ne pas faire du 100 % sur mesure ?

Les portfolios d’artistes n’ont pas besoin de CMS complexes ni de plugins, qui rendent souvent les sites plus lourds et fragiles. En créant uniquement ce qui est nécessaire, on obtient un site léger, stable et durable, bien moins susceptible de “se casser”.

Pourquoi parle-t-on de « frais d’assistance à la création » et non de « frais de création » ?

Ce service est conçu pour réduire la dépendance entre vous et moi dans la gestion à long terme du site. Ainsi, tous les fichiers du code et le compte GitHub sont entièrement votre propriété dès le départ.

Vous êtes responsable de la direction artistique de votre site, tandis que moi, je fournis les propositions et le support technique nécessaires pour réaliser cette vision.

Autrement dit, le site n’est pas quelque chose que je « crée pour vous », mais une co-création entre vous et moi. C’est pourquoi ce coût est défini comme des « frais d’assistance à la création » et non comme des frais de création classiques.

制作と支払いの流れ

このサービスは、できる限りスムーズで明確なプロセスで進行します。最初のヒアリングからサイト公開までの流れは以下の通りです。

1. 方向性のヒアリング

まず、あなたのポートフォリオサイトの方向性を一緒に確認します。 現在の立ち位置、見せたいイメージ、カテゴリ構成、全体の世界観などを共有し、最適なレイアウトやサイト構造を決めていきます。

2. 素材（画像・テキスト）の送付

ヒアリング後、クライアント側で掲載したい写真・キャプション・クレジットなどを送っていただきます。 すでに構成が固まっている場合はそのまま、まだ整理が必要な場合は流れを一緒に整えます。

3. ウェブ制作開始

素材がそろい次第、私がコーディングとレイアウト調整を開始します。 Overview・Lightbox・カテゴリ構成・必要なテキストページなどを組み込み、あなた専用のポートフォリオサイトを構築していきます。

4. テストリンクで仮公開

完成後、まずは GitHub Pages 上にテスト用リンクとして仮公開します。 実際にスマホ・タブレット・PC で動作を確認し、画像の並びやキャプションなどに問題がないか、一緒にチェックします。

5. クライアントの最終確認と承認

修正点があれば反映し、最終版が問題なければ「承認」をいただきます。 このタイミングで制作が完了したことになり、支払いに進みます。

6. 支払い

テストリンクを共有したあと、スマートフォン・PC などでゆっくり内容を確認していただきます。 内容に問題がなければ「承認」をいただき、制作補助費（1200€）をお支払いください。

7. 支払い確認後、ドメイン接続・本公開

支払い確認後、あなたが指定した独自ドメインにサイトを紐づけ、本公開します。 公開後は正式なポートフォリオとして機能し、この時点で制作工程は完了です。

公開後、数日経ってから気づくような軽微な不具合（画像の表示崩れ、小さなレイアウト調整など）があれば、 これらは無償修正にて対応しますのでご安心ください。

FAQ

これはフルカスタムですか？

基本はセミカスタムサイトです。ただし最終的な仕上がりは、ほぼフルカスタムに近いクオリティになります。

フルカスタムとセミカスタムの違いはなんですか？

一般的なセミカスタムは、既存テンプレートや CMS（Wix・WordPress など）をベースに、色や配置だけを調整する方式です。 私のサイトはテンプレートを使用せず、レイアウトや画像の見せ方までゼロから作る “フルカスタム寄り” の制作です。

なぜフルカスタムにしないのですか？

アーティストのポートフォリオでは複雑な CMS やプラグインは不要であり、むしろ重く壊れやすくなります。 必要な部分だけをゼロから設計することで、軽くて壊れにくい、長期保守に強いサイトを実現しています。

なぜ「制作費」ではなく「制作補助費」なのですか？

このサービスは、お客様と私の間に生まれるサイト運営上の依存関係を減らすことを目的としています。 そのため、サイトのコードファイルと GitHub アカウントはすべてお客様自身の所有物となる設計です。

お客様はサイト制作のアートディレクションを担当し、 私はそのビジョンを実現するための提案と技術的サポートを行います。

つまり、サイト制作は「私が作るもの」ではなく、 お客様と私による共同制作という位置づけです。 そのため、この料金は “制作費” ではなく “制作補助費” としています。

サイトの所有者は誰ですか？

常にあなたです。GitHub のアカウントもコードもあなた名義です。

2年後の更新は必須ですか？ また、更新時期はいつですか？

いいえ、更新は任意です。更新しなくてもサイトはそのまま残ります。 更新のご案内は、サポート期間の終了1ヶ月前にお知らせします。

無料更新の範囲は？

週2回までの小さな更新（画像追加・差し替え・キャプション修正など）です。

Natsuki がこのサービスをやめることはありますか？

はい、あります。 私の本業はメイクアップアーティストで、将来的に忙しくなり継続が難しくなる可能性があります。 終了する場合は 6ヶ月前までに必ずお知らせします。

では、サイト納品後すぐにこのサービスをやめてしまう可能性はありますか？

いいえ、その心配はありません。 納品後の初回2年間のサポートは必ず履行します。 これはサービスの根幹であり、私自身に重大な事情（事故・病気など）がない限り、途中で放棄することはありません。 初回の2年間については、安心してご利用いただけます。

サイトはあなたの GitHub 上にあり、完全にあなたの所有物です。 そのまま使い続けることも、他のデザイナーに引き継ぐこともできます。

他のデザイナーに引き継いだり、コードを修正してもらっても大丈夫ですか？

はい、大丈夫です。サイトデータはあなたの所有物なので、他のデザイナーに依頼しても問題ありません。
