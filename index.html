About GitHub – How It Works & Who Owns the Site

GitHub is a Microsoft-owned platform and one of the largest code-hosting services in the world. It is where millions of developers and designers store, share, and publish their projects. In terms of reliability, speed, and stability, it is one of the strongest free hosting environments available today.

What is “hosting”?

Hosting simply means placing your site files (HTML / CSS / JavaScript, etc.) on an online server so anyone can access your website. Many people imagine paid servers or monthly contracts, but GitHub Pages provides hosting completely for free. Creating an account is also free.

Why many professional designers use GitHub today

In recent years, more and more professional web designers have started building client websites on GitHub Pages. The reason is simple — it meets all the essential conditions:

  • Fast
  • Stable
  • Secure
  • No running costs
  • Hard to break
  • No dependency (the owner is always you)

How this differs from design studios and standard freelancers

Even if a designer uses GitHub, most studios or freelancers will still charge “maintenance fees” or “management fees” every month or every year.

But in my system, the site is built directly inside your own GitHub account, and the code is 100% owned by you. As long as your GitHub account exists, the site remains your asset forever.

Why this method is ideal for artist portfolios

Portfolio sites do not need heavy features or large servers. In reality, the best portfolio is one that is:

  • Lightweight
  • Stable and hard to break
  • Fast
  • Easy to maintain
  • Independent of any designer or company

Using this structure allows me to create a long-lasting, agency-level portfolio site with minimal cost and maximum control for the artist.

Market Facts – What Website Production Usually Costs

At first glance, 1200€ may seem high. But when you look at the actual market rates for portfolio websites, this price is actually on the lower end.

Approximate breakdown when hiring a professional web designer

For a simple hair & makeup portfolio site — for example, an overview page plus a lightbox gallery — the typical cost from a freelance designer in Europe looks something like this:

  • Consultation & structure planning: 200–400€ (site map, deciding what goes where)
  • Design (top page + overview + basic info page): 400–700€
  • Coding (HTML / CSS / JavaScript, responsive layout): 400–800€
  • Overview gallery + lightbox implementation: 300–500€
  • Server setup, domain connection, final checks: 150–300€

Altogether, this usually comes to around 1500–2700€, depending on the project and designer.

Monthly “maintenance fees”

Many designers and production companies also charge monthly or annual fees for:

  • Server & domain management
  • Security updates or backups
  • Small text or image updates

These maintenance plans typically cost 30–100€ per month, sometimes more depending on the provider.

In reality, website creation and maintenance add up quickly — especially for artists, whose portfolio is part of their professional identity and daily work.

That is why the one-time creation fee of 1200€ (including two years of support) is designed to be a realistic and predictable option. Everything essential is included upfront, with no hidden running costs.

Héberger un site signifie simplement placer vos fichiers (HTML / CSS / JavaScript, etc.) sur un serveur en ligne pour que tout le monde puisse y accéder.
Beaucoup imaginent des serveurs payants ou des abonnements mensuels,
mais GitHub Pages offre l’hébergement entièrement gratuitement.
La création d’un compte est gratuite également.

Ces dernières années, de plus en plus de webdesigners professionnels ont commencé à créer des sites clients avec GitHub Pages.

GitHub について – 仕組みと「サイトの所有者」

GitHub は Microsoft 傘下のプラットフォームで、世界最大級のコードホスティングサービスの一つです。 何百万人もの開発者やデザイナーが、ここでプロジェクトを保存・共有・公開しています。 信頼性、速度、安定性という点で、現在利用できる「無料ホスティング環境」の中でも 非常に強力な選択肢の一つです。

「ホスティング」とは？

ホスティングとは、サイトのファイル（HTML / CSS / JavaScript など）を オンライン上のサーバーに置き、誰もがブラウザからアクセスできる状態にすることです。
一般的には「有料サーバー契約」や「月額料金」をイメージする方が多いですが、 GitHub Pages を使えば、ホスティングは完全に無料です。
アカウントの作成も無料です。

なぜ今、多くのプロのデザイナーが GitHub を使うのか

ここ数年、多くのプロのウェブデザイナーが クライアントサイトの公開に GitHub Pages を使うようになってきました。
理由はとてもシンプルで、ポートフォリオに必要な条件をすべて満たしているからです。

  • 表示が速い
  • 安定している
  • セキュリティ面で安心
  • ランニングコストがかからない
  • ちょっとした操作では壊れにくい
  • 依存関係がない（所有者は常に「自分」）

制作会社や一般的なフリーランスとの違い

デザイナーが GitHub を使っていたとしても、 多くの制作会社やフリーランスは「保守費」や「管理費」といった名目で 月額または年額の料金を請求し続けます。

一方この仕組みでは、サイトは最初から あなた自身の GitHub アカウントの中に構築され、 コードも含めて 100％あなたの所有物になります。
あなたの GitHub アカウントが存在する限り、そのサイトはあなたの資産として残り続けます。

なぜアーティストのポートフォリオに向いているのか

ポートフォリオサイトに「重い機能」や「大規模なサーバー」は基本的に必要ありません。
実際に最も理想的なポートフォリオとは、次のようなものだと考えています。

  • 軽い（表示が速い）
  • 壊れにくく、安定している
  • 管理が簡単
  • 特定のデザイナーや会社に依存しない

この構造を採用することで、 コストを最低限に抑えつつ、アーティスト側のコントロールを最大化したまま、 長く使える「エージェンシーレベル」のポートフォリオサイトを作ることができます。

制作とお支払いの流れ

このプロジェクトは、シンプルで分かりやすいステップで進みます。
最初のヒアリングから公開までの流れは、次のようになります。

1. 方向性のミーティング

まずは、ポートフォリオサイト全体の方向性について話し合います。 現在の活動状況、見せたい画像、必要なカテゴリ、サイトのトーンなどを整理し、 一緒に最適な構成とレイアウトを決めていきます。

2. 画像とテキストの受け取り

方向性が固まったら、写真、キャプション、クレジット、プロフィール文など、 必要な素材を送っていただきます。 すでに整理されている場合はそのまま使い、まだバラバラな場合は、一緒に整理していきます。

3. サイト制作（デザイン & コーディング）

素材が揃ったら、実際の制作に入ります。 Overview ギャラリーやライトボックス、カテゴリ構造、必要なテキストページなど、 あなた専用のポートフォリオサイトの形を作っていきます。

4. テストリンク（プレビュー）

制作が一通り完了したら、GitHub Pages を使ってテスト用のリンクを公開します。 スマホ・タブレット・パソコンなど、ご自身のデバイスで表示をチェックしてもらい、 画像の流れやレイアウト、テキストの内容を確認していただきます。

5. 最終調整 & 承認

修正したい点があれば、この段階で反映します。 最終版に問題がなくなったところで、サイトの内容に「OK」をいただきます。 ここまでで制作工程が完了し、次がお支払いのステップになります。

6. お支払い

テストリンクを実際にご自身の環境で確認し、 「この内容で公開して大丈夫」となったタイミングで、 制作サポート費（1200€）のお支払いをお願いする形になります。

7. ドメイン接続 & 正式公開

入金が確認できたら、ご希望の独自ドメインとサイトを接続し、 正式なポートフォリオサイトとして公開します。

公開後数日の間に、 デバイスの違いなどで小さなズレ（余白、画像の位置など）が見つかる場合がありますが、 そうした軽微な調整は 無料で対応 します。 静的サイトでは、ごく自然に起こりうる範囲のものです。

キャンセル・返金ポリシー

このサービスは、信頼している友人に向けた「成果ベース」のサポートプロジェクトです。
まずテストリンクで完成版を確認してもらい、そのうえでお支払いに進んでいただきます。

キャンセルについて

制作の途中で方向性が大きく変わったり、続行が難しくなった場合は、 どの段階でもキャンセルしていただいて構いません。 その際は、できるだけ早いタイミングで状況を共有してもらえると助かります。

承認後の返金について

テストリンクを確認し、「この内容で大丈夫」と承認いただいた時点で、 制作は完了とみなされます。
その後の返金は基本的に受け付けられませんが、 私のミスによる、致命的かつ修正不能なコード上のエラー がある場合のみ、 例外的に返金を検討します。

公開から数日以内に見つかるような小さな不具合 （軽微なレイアウト調整や画像位置の微修正など）は、 静的サイトではよくある範囲のものですので、 これらは 無料で修正 します。

外部要因について

iOS やブラウザのアップデート、GitHub の仕様変更、 特定デバイス固有の挙動など、外部要因による変化が原因の問題については、 返金の対象外 とさせていただきます。

そのような外部要因によるトラブルが発生した場合も、 可能な範囲で状況を確認し、必要に応じて調整や修正の提案を行います。 ただし、作業のボリュームによっては、追加費用が発生する場合があります。

FAQ

Q. 他のデザイナーにコードを引き継ぐことはできますか？

A. はい、可能です。サイトはあなたのものなので、 どのデザイナーと一緒に更新をするかは、自由に選んでいただけます。

Q. 納品後に、他の人にコードを修正してもらってもいいですか？

はい、可能です。ただし第三者がコードを書き換えた場合、 私が用意した構造とは異なる状態になる可能性があります。
その場合、サポート期間中であっても、同じ条件での更新や修正の対応が難しくなる、または継続できなくなる可能性がございます。
ご不安な場合は、作業前に一度ご相談いただけると安心です。

ここでいう「コードの維持・更新」とは、画像の追加・差し替えや、キャプションなどの軽いテキスト変更のことを指します。 新しいページの追加や新機能の実装など、大幅な構造変更を含むものではありません。

Q. 少人数制とはどれくらいの人数ですか？

A. 現時点では10名ほどを想定していますが、状況に応じて無理のない範囲で増減する可能性があります。あらかじめご了承ください。

Q. これはフルカスタムサイトですか？

A. 厳密には「セミカスタム」ですが、最終的な見た目やクオリティは フルカスタムにかなり近い仕上がりになります。

Q. フルカスタムとセミカスタムの違いは何ですか？

A. 一般的なセミカスタムサイトは、Wix や WordPress のような既存のテンプレートや CMS をベースに、 色やフォント、簡単なレイアウトだけを調整するものが多いです。
私の方法ではテンプレートを一切使わず、レイアウト・画像構成・余白・見せ方を 最初から設計していくため、結果として「ほぼフルカスタム」に近いサイトになります。

Q. なぜ最初からフルカスタムにしないのですか？

A. アーティストのポートフォリオには、重いCMSや複雑なプラグインは基本的に不要で、 むしろそれらが原因で、サイトが重くなったり壊れやすくなったりすることがあります。
必要な部分だけを手作りすることで、軽くて安定した、長期運用向きのポートフォリオになります。

Q. なぜ「制作費」ではなく「制作サポート費」と書いてあるのですか？

A. このサービスは、あなたと私のあいだに「不要な依存関係」を作らないことを 大きなテーマにしています。
そのため、コードファイルや GitHub アカウントは、最初から すべてあなたの所有物 という前提で進めます。

サイトのアートディレクションはあなた自身が担い、 私はそのビジョンを形にするための提案と技術サポートを提供する立場です。

つまりこのサイトは、「私が作ってあげるもの」というより、 あなたと私の共同制作 というイメージに近いと思っています。
そのため、名目として「制作サポート費」という表現を使っています。

Q. サイトの所有者は誰になりますか？

A. サイトの所有者は、あくまであなたです。 GitHub アカウントも、コード一式も、あなたの名義で管理されます。

Q. 2年ごとの更新は必須ですか？タイミングは？

A. 更新は完全にオプションです。 更新しなくてもサイトはそのままオンラインに残ります。
サポート期間の終了が近づいたら、1か月前を目安に私の方からお知らせします。

Q. 無料更新には何が含まれますか？

A. 週2回までの小さな更新（画像の追加・差し替え、クレジットやテキストの微調整など）が含まれます。

Q. 将来的に、Natsuki がこのサービスをやめてしまう可能性はありますか？

その場合でも、少なくとも 6か月前 には事前にお伝えします。

初期2年のサポートについては、安心していただいて大丈夫です。

面倒な技術的作業はこちらで対応しますので、コードに触れる必要はありません。

Q. 新しい写真や更新内容はどうやって送ればいいですか？

A. DM やメールなど、いつも使っている連絡手段で大丈夫です。 画像・キャプション・クレジットなどを送ってもらえれば、 「週2回までの更新」の範囲で反映していきます。

Q. 画像点数に制限はありますか？

どのくらいのボリュームが最適かは、一緒に相談しながら決めていきます。

Q. SEO 対策はどうなっていますか？

A. アーティストのポートフォリオに対して、 いわゆる「攻めの SEO（検索上位表示を狙う施策）」は必須ではないと考えています。

このサイトの目的は、大勢に向けて広く見つけてもらうことではなく、 あなたが本当に見せたい相手に、きちんと作品を届けること です。
